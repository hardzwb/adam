# 下 RING 模块方案文档

## 1. 设计目标

下 RING 模块用于连接上游 credit 协议 CHI 通道和下游 valid/ready 协议 CHI 通道。模块包含四个彼此独立的通道：

- `req`
- `rsp`
- `dat`
- `snp`

每个通道内部配置一组 FIFO。FIFO 当前空位数作为 credit 返回给上游；下游侧使用标准 `vld/rdy` 握手。

## 2. 顶层文件

RTL 文件位于：

- `rtl/chi_down_ring.sv`：四通道顶层封装。
- `rtl/chi_credit_fifo.sv`：单通道 credit FIFO。

## 3. 接口协议

### 3.1 上游输入侧

上游每个通道提供：

- `*_in_vld_i`
- `*_in_i`
- `*_in_credit_o`

输入侧没有 `rdy`。上游必须根据 `*_in_credit_o` 控制发送，只有 credit 大于 0 时才能继续发送数据。若上游在 FIFO 已满且没有同周期出队空间时仍拉高 `*_in_vld_i`，模块会拉高对应的 `*_overflow_o` 一个周期，用于错误观测。

### 3.2 下游输出侧

下游每个通道提供：

- `*_out_vld_o`
- `*_out_o`
- `*_out_rdy_i`

当 `*_out_vld_o && *_out_rdy_i` 为 1 时，完成一次输出传输。

## 4. 参数和宏

通道位宽通过宏传入，顶层也保留参数覆盖能力：

| 宏 | 默认值 | 说明 |
| --- | ---: | --- |
| `CHI_REQ_W` | 128 | req 通道位宽 |
| `CHI_RSP_W` | 64 | rsp 通道位宽 |
| `CHI_DAT_W` | 512 | dat 通道位宽 |
| `CHI_SNP_W` | 128 | snp 通道位宽 |

FIFO 深度通过以下宏配置：

| 宏 | 默认值 | 说明 |
| --- | ---: | --- |
| `CHI_REQ_FIFO_DEPTH` | 4 | req FIFO 深度 |
| `CHI_RSP_FIFO_DEPTH` | 4 | rsp FIFO 深度 |
| `CHI_DAT_FIFO_DEPTH` | 4 | dat FIFO 深度 |
| `CHI_SNP_FIFO_DEPTH` | 4 | snp FIFO 深度 |

credit 位宽由深度自动推导：

```systemverilog
CREDIT_W = (DEPTH <= 1) ? 1 : $clog2(DEPTH + 1)
```

## 5. FIFO 行为

每个 FIFO 维护当前占用量 `occ_q`：

```text
credit = DEPTH - occ_q
```

复位后 FIFO 为空，credit 等于 FIFO 深度。

FIFO 支持 bypass。当 FIFO 为空、输入 `vld` 有效且下游同周期 `rdy` 有效时，数据不写入存储阵列，而是直接从输入侧透传到输出侧：

```text
fifo empty + in_vld + out_rdy => bypass transfer
```

bypass 场景下 FIFO 占用量保持为 0，因此不会消耗额外 credit。

## 6. 背压与保护

由于输入侧没有 `rdy`，模块不能通过握手阻止上游继续发送。系统正确性依赖上游遵守 credit：

```text
上游可发送条件：credit > 0
```

若上游违反 credit 约束，FIFO 不会覆盖已有数据，对应通道 `*_overflow_o` 拉高一个周期。

## 7. 通道独立性

四个通道完全独立：

- 独立 FIFO 深度。
- 独立 credit。
- 独立 valid/ready 输出。
- 独立 overflow 观测。

任一通道阻塞不会影响其他通道继续传输。

## 8. 集成建议

实际集成时建议在编译命令或上层 package 中定义真实 CHI 位宽宏，例如：

```systemverilog
`define CHI_REQ_W 123
`define CHI_RSP_W 45
`define CHI_DAT_W 512
`define CHI_SNP_W 88
```

若项目已有统一 CHI payload typedef，也可以在顶层实例化时通过参数覆盖位宽，而不依赖默认宏。
