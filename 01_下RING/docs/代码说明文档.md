## 2. `chi_down_ring.sv`

`CHI_DOWN_RING` 是顶层模块，负责将四个 CHI 通道分别实例化为独立 FIFO。

### 2.1 参数

顶层参数分为两类：

- 通道位宽：`REQ_W`、`RSP_W`、`DAT_W`、`SNP_W`
- FIFO 深度：`REQ_FIFO_DEPTH`、`RSP_FIFO_DEPTH`、`DAT_FIFO_DEPTH`、`SNP_FIFO_DEPTH`

默认位宽和深度来自宏定义。若外部没有定义宏，文件内提供可编译的默认值。

### 2.2 端口

每个输入通道包含：

```systemverilog
input  logic             <ch>_in_vld_i
input  logic [W-1:0]     <ch>_in_i
output logic             <ch>_in_credit_o
```

每个输出通道包含：

```systemverilog
output logic             <ch>_out_vld_o
output logic [W-1:0]     <ch>_out_o
input  logic             <ch>_out_rdy_i
```

每个通道额外输出一个 overflow 标志：

```systemverilog
output logic             <ch>_overflow_o
```

该信号用于发现上游违反 credit 发送的情况。

### 2.3 实例化关系

顶层实例化四个 `CHI_CREDIT_FIFO`：

- `U_REQ_FIFO`
- `U_RSP_FIFO`
- `U_DAT_FIFO`
- `U_SNP_FIFO`

四个实例仅参数不同，控制逻辑完全一致。

## 3. `chi_credit_fifo.sv`

`CHI_CREDIT_FIFO` 是单通道 FIFO，提供 credit 输入协议到 valid/ready 输出协议的转换。

### 3.1 主要参数

```systemverilog
parameter int DATA_W
parameter int DEPTH
```

### 3.2 状态寄存器

FIFO 内部使用以下状态：

```systemverilog
mem_q      // 数据存储阵列
rd_ptr_q   // 读指针
wr_ptr_q   // 写指针
occ_q      // 当前占用数量
credit_cnt_q // 待发放 credit 数量
```

`occ_q == 0` 表示空，`occ_q == DEPTH` 表示满。

### 3.3 输出逻辑

当 FIFO 非空时，输出来自 `mem_q[rd_ptr_q]`：

```systemverilog
out_vld_o  = 1'b1
out_data_o = mem_q[rd_ptr_q]
```

当 FIFO 为空时，输出直接观察输入：

```systemverilog
out_vld_o  = in_vld_i
out_data_o = in_data_i
```

因此在空 FIFO 且下游 ready 的情况下可以零额外周期通过。

### 3.4 入队、出队和 bypass

关键控制信号：

```systemverilog
bypass_xfer_w  = fifo_empty_w && in_vld_i && out_rdy_i
pop_entry_w    = !fifo_empty_w && out_rdy_i
accept_input_w = in_vld_i && (!fifo_full_w || pop_entry_w || bypass_xfer_w)
push_entry_w   = accept_input_w && !bypass_xfer_w
```

含义如下：

- `bypass_xfer_w`：空 FIFO 下的直接传输。
- `pop_entry_w`：已有 FIFO entry 被下游接收。
- `accept_input_w`：本周期输入可被接收。
- `push_entry_w`：输入需要真实写入 FIFO 存储阵列。

### 3.5 credit 生成

FIFO 内部使用 `credit_cnt_q` 记录待发放的 credit 数量。复位释放后，`credit_cnt_q` 初始为 `DEPTH`。每发出一个 `credit_o` 脉冲，credit 总数减 1；每次下游通过 `out_vld_o && out_rdy_i` 接受一个 beat，credit 总数加 1。发放和归还同周期发生时，credit 总数保持不变。

### 3.6 overflow 生成

当 FIFO 已满、没有出队空间、也不是 bypass 场景，而上游仍然发送有效数据时，拉高 overflow：

```systemverilog
overflow_o = in_vld_i && fifo_full_w && !pop_entry_w && !bypass_xfer_w
```

overflow 不参与恢复控制，只作为协议违例观测信号。

## 4. 验证关注点

建议后续补充 testbench 覆盖以下场景：

- 复位后连续发放 `DEPTH` 个初始 credit 脉冲。
- 空 FIFO 下输入与输出 ready 同周期有效，数据 bypass。
- 下游 backpressure 时数据进入 FIFO 并保持顺序。
- FIFO 满且没有出队时不继续发放 credit。
- FIFO 满且上游继续发送时 overflow 拉高。
- 四个通道同时工作时互不影响。
